C++标准库 — IO库

IO类

[TOC]

头文件 类型
iostream istream, wistream从流读数据 ostream, wostream从流写数据; stream,wstream从文件读写数据
fstream ifstream, wifstream从文件读数据 ;ofstream, wofstream从文件写数据; fstream,wfstream从文件读写数据
sstream istringstream, wistringstream从string读数据; ostringstream, wostringstream从string写数据; stringstream, wstringstream读写数据

上面的带w前缀的类型都是用来处理wchar_t类型的数据的。

ifstream和istringstream继承自istream, ofstream和ostringstream继承自ostream;

IO对象不能进行拷贝或赋值

因为IO对象不能进行拷贝和赋值操作，下面的行为全都是错的：

ofstream out1, out2;
out1 = out2;
ofstream print(ofstream);
out2 = print(out2);

正确的使用IO对象的方式是通过引用的方式对其进行传递和返回。

条件状态

strm::iostate strm是一种IO类型，iostate是一种与机器无关的类型提供了表达条件状态的完整功能
strm::badbit 指出流已经崩溃
strm::failbit 指出一个IO操作失败了
strm::eofbit 流到了文件结束
strm::goodbit 指出流未处于错误状态，应保证为0
s.eof() 若eofbit置位则返回true
s.fail() 返回failbit是否置位
s.bad() 返回badbit是否置位
s.good() 若goodbit处于有效状态返回true
s.clear() 将流中所有状态复位，将流设为有效
s.clear(flags) 根据flags复位
s.setstate(flags) 根据flags置位
s.rdstate() 返回s的当前状态，类型为strm::iostate

badbit表示系统级的错误，被置位后流就不能使用了。

failbit表示发生可恢复错误，比如期望读入数值却读入了字符，流还可以被使用。goodbit为0表示发生错误。如果badbit,failbit和eofbit任何一个被置位，则条件检测状态会失败。

输出缓冲

每个输出流都管理着一个缓冲区，os<<"Please enter a value:"这个操作可能会马上显示也可能被存起来直到缓冲刷新才输出。

有很多操作会导致缓冲区被刷新，比如:

  1. 程序结束

    注意，若程序崩溃，输出缓冲区不会被刷新，它的输出数据很可能停留在缓冲区等待打印。所以当调试一个已经崩溃的程序时，需要确认已经输出的数据确实已经刷新了，否则，可能将大量时间浪费在代码为什么没有执行上。

  2. 缓冲区满

  3. endl,flush, ends操作符

    • cout<<"hi"<<endl;:输出内容和换行，然后刷新缓冲区
    • cout<<"hi"<<flush;:输出内容然后刷新缓冲区
    • cout<<"hi"<<ends;输出内容和一个空字符，然后刷新缓冲区

  4. 使用unitbuf操作符设置内部状态，如果流设置了这个操作符，则该流的写入操作是立即刷新的，比如cerr默认设置了unitbuf因此对它的写入操作会马上刷新

    cout << unitbuf;    //所有输出操作都会立即刷新缓冲区
cout << nounitbuf;    //回到正常缓冲方式

    ​

  5. 如果一个流和另一个流相关联，当被关联的流被读写时，关联流缓冲区也会被刷新。比如cin和cerr都被关联到了cout所以，只要读cin或写cerr都会导致cout刷新

文件输入输出

头文件fstream定义了三个类型来支持文件IO:

  • ifstream:从给定文件读数据
  • ofstream:写数据到给定文件
  • fstream:读写给定文件

除了通用的<<, >>, getline等操作外文件IO有一些特殊的方法:

操作 说明
fstream fstrm; 创建一个未绑定的文件流，fstream是头文件fstream定义的一个类型
fstream fstrm(s); 创建一个fstream并打开名为s的文件。s可以使string 类型或者c风格字符串指针，构造函数是explicit的。默认文件模式依赖于fsteam的类型
fstream fstrm(s, mode); 创建一个fstream，并按指定模式打开文件s。
fstrm.open(s); 打开名为s的文件，并将文件与fstream绑定。
fstrm.close() 关闭与fstrm绑定的文件
fstrm.is_open() 返回一个bool值，指出关联的文件是否被成功打开且尚未关闭

下面的例子以传递给main的参数来制定文件进行读写

int main(int argc, string argv[])
{
  ifstream input(argv[1]);
  ofstream output(argv[2]);
  Sales_data total;
  if (read(input, total)) {
    Sales_data trans;
    while(read(input, trans)) {
      if (total.isbn() == trans.isbn()){
        total.combine(trans);
      } else {
        print(output, total) << endl;
        total = trans;
      }
    }
    print(output, total) << endl;
  } else {
    cerr << "No data?!" << endl;
  }
}

open 和 close成员函数

可以先定义一个空的文本流对象，然后调用open将它与文件关联起来，如果调用open失败，failbit会失败，所以对open进行检查很有必要:

ifstream in(file);
ofstream out(file);
out.open(ifile + ".copy");
if (out) {//对open进行检查

}

打开另一个文件之前需要调用close将原先文件流关闭

out.close();
out.open(file);

自动构造和析构

当fstream对象离开其作用域时，与之关联的文件会被自动关闭:

for (auto p = argv + 1; p != argv + argc; ++p) {
  ifstream input(p);
  if (input) {
    process(input);
  } else 
      cerr << "counldn't open: " + string(*p);
}

上面的程序循环遍历参数提供的文件列表，在每次循环结束后文件都会被自动关闭而不需要使用close显式关闭。

文件模式

模式 说明
in 读模式打开,只能对ifstream, fstream设置
out 写模式打开，只能对ofstream,fstream设置
app 每次写操作定位到文件 末尾，只要没有设置trunc模式就可以设定
ate 打开文件后立即定位到文件末尾，可用于任意文件流对象，可以和任何模式组合
trunc 截断文件，只有out模式被设置时才可以设定，默认是这样的截断模式
binary 以二进制方式打开进行IO，可以用于任意文件流对象，可以和任何模式组合

默认模式

流 默认模式
ifstream in
ofstream out
fstream in+out

out模式会丢弃打开文件的内容，为了阻止这个清空操作，需要同时指定app模式：ofstream app("file", ofstream::out | ofstream::app)

string 流

sstream头文件定义了三个类型来支持内存IO:

类型 作用
istringstream 从string读数据
ostringstream 写数据到string
stringstream 读写string

sstream除了定义通用操作之外，还定义了如下操作

操作 说明
sstream strm strm是未绑定的stringstream对象。sstream是头文件sstream中定义的一个类型
sstream strm(s) strm保存着string s的一个拷贝，这个构造函数是explicit的
strm.str() 返回strm保存的string的拷贝
strm.str(s) 将string s拷贝到strm中，返回void
strm.clear() 清空当前流
