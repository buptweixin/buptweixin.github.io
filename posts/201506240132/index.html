C++ primer笔记 — 类


[TOC]

定义抽象数据类型

  1. 非静态成员函数的参数列表后面加const表示成员函数的this指针为const指针，即在该成员函数中修改它所在类的成员的操作是不被允许的

      struct Sales_data
  {
    std::string isbn() const {return bookNo;}
  }

       这样的成员函数称为常量成员函数

  2. 成员函数

    类的成员函数必须在类内声明，但是可以在类内或者类外定义，如果定义在类内则该成员函数是隐式的inline函数。

    类外定义的成员函数：

      double Sales_data::avg_price() const {
    if (units_sold)
      return revenue/units_sold;
    else
      return 0;
  }

  1. this

    成员函数中有一个隐式变量this,它指向当前的对象，可以通过this.xxx调用类成员或者return *this;返回当前对象。

  1. 定义类相关的非成员函数

    如果函数在概念上属于类但是不定义在类中，则它一般与类声明(不是定义)在同一个文件中

构造函数

构造函数名和类名一致，没有返回类型，它控制着类的初始化过程。

构造函数不能声明为const，当我们创建一个类的const对象时，直到构造函数完成初始化过程，对象才真正取得常量属性。所以，构造函数在const对象的构造过程中可以向其写值。

默认构造函数

当我们没有显式地声明构造函数时，编译器会通过默认构造函数来初始化，这个默认构造函数也叫作合成构造函数，如果存在类内初始值(如Sales_data中的bookNo等)则用它来初始化成员，否则默认初始化成员。

如果一个构造函数为所有参数都提供了默认实参，则它实际上也定义了默认构造函数

构造函数的初始值有时必不可少

如果成员是const、引用或属于某种没有定义默认构造函数的类类型的话，我们必须对其初始化：

class ConstRef {
  public:
      ConstRef(int ii)
    {
         i = ii;
          ci = ii; //错误，不能给const赋值
          ri = ii; //错误，ri没有被初始化
    }
  private:
      int i;
      const int ci;
      int &ri;
}

初始化const、引用的唯一机会是通过构造函数初始值：

ConstRef(int ii): i(ii), ci(ii), ri(ii) {}

成员初始化顺序

不管构造函数初始值顺序是怎么样的，实际上初始化的顺序和成员声明出现的顺序一致：

Class X {
  int i;
  int j;
 public:
     // 未定义的: i在j之前被初始化
     X(int val): j(val), i(j) {}
}

委托构造函数

使用所属类的其他构造函数执行自己的初始化过程

class Sales_data {
  public:
      // 非委托构造函数
    Sales_data(const std::string &s, const unsigned n, double p):
        bookNo(s), units_sold(n), revenue(n * p) {}
      // 委托构造函数
      Sales_data(): Sales_data("", 0, 0) {}
      Sales_data(string s): Sales_data(s, 0, 0) {}
    Sales_data(std::istream &): Sales_data() {read(is, *this); } // 外部定义的构造函数
}

隐式类型转换

如果构造函数只接受一个实参，则它实际上定义了将实参类型转换成类类型的隐式转换，比如我们可以使用string或istream替代Sales_data

string null_book = "9-999-9999-9";
item.combine(null_book);

如果不想要这种隐式类型转换，可以使用explicit抑制它

explicit抑制隐式类型转换

explicit只能在类内声明构造函数时使用。

class Sales_data {
  public:
    Sales_data() = default; //指定保留默认构造函数
    explicit Sales_data(const std::string &s): bookNo(s) {}
    Sales_data(const std::string &s, const unsigned n, double p):
        bookNo(s), units_sold(n), revenue(n * p) {}
    explicit Sales_data(std::istream &); // 外部定义的构造函数
}

使用static_cast显式类型转换

可以使用static_cast 使用explicit抑制的类型转换

item.combine(static_cast<Sales_data> (cin));

标准库中含有显式构造函数的类

  • 接受一个const char*的string构造函数，不是explicit的
  • 接受一个容器参数的vector,是explicit的

为Sales_data定义构造函数

struct Sales_data {
    Sales_data() = default; //指定保留默认构造函数
    Sales_data(const std::string &s): bookNo(s) {}
    Sales_data(const std::string &s, const unsigned n, double p):
        bookNo(s), units_sold(n), revenue(n * p) {}
    Sales_data(std::istream &); // 外部定义的构造函数
}

Sales_data::Sales_data(std::istream &is)
{
    read(is, *this);
}

拷贝、赋值和析构

类中 如果包含vector或string成员则其拷贝、赋值和销毁的合成版本能够正常工作否则需要自己定义。

聚合类

满足下列条件:

  • 所有成员都是public的
  • 没有定义任何构造函数
  • 没有类内初始值
  • 没有基类、virtual函数

则它是聚合类，用户可以直接访问它的成员并且使用{}的初始化语法。

struct Data {
  int ival;
  string s;
};

Data val1 = {0, "Annel"};

字面值常量类

constexpr构造函数

访问控制和封装

  • public

    程序内可访问, 用于定义类的接口

  • private

    类内可访问，封装(隐藏)类的实现细节。

封装的优点:

  1. 确保用户代码不会无意间破坏封装对象的状态；
  2. 被封装的类的具体实现细节可以随时改变而无需调整用户级别代码。

使用class关键字

class和struct仅仅形式化有所不同，另外struct默认访问权限是public，而class是private的

class Sales_data {
public:
    Sales_data() = default;
    Sales_data(const std::string &s): bookNo(s) {}
    Sales_data(const std::string &s, const unsigned n, double p):
        bookNo(s), units_sold(n), revenue(n * p) {}
    Sales_data(std::istream &);
    string isbn() const { return bookNo;}
    Sales_data& combine(const Sales_data&);

private:
    string bookNo;
    unsigned units_sold = 0;
    double revenue = 0.0;
}

Sales_data add(const Sales_data &lhs, const Sales_data &rhs)
{
    Sales_data sum  = lhs;
    sum.combine(rhs);
    return sum;
}

std::istream &read(std::istream &is, Sales_data &item)
{
    double price = 0;
    is >> item.bookNo >> item.units_sold >> price;
    item.revenue = price * item.units_sold;
    return is;
}

std::ostream &print(std::ostream &os, const Sales_data &item)
{
    os << item.isbn() << " " << item.units_sold << " " << item.revenue;
    return os;
}

Sales_data& Sales_data::combine(const Sales_data& rhs)
{
    units_sold += rhs.units_sold;
    revenue += rhs.revenue;
    return *this;
}

友元

在Sales_data的例子中，read、print、add函数虽然是接口的一部分但不是类的成员，这样他们就无法访问声明为private的数据成员而造成无法编译，为了解决这个问题，需要使用”友元”。

可以在类内部通过friend关键字声明友元

class Sales_data {
friend Sales_data add(const Sales_data &, const Sales_data &);
friend std::istream &read(std::istream &is, Sales_data &item);
friend std::ostream &print(std::ostream &os, const Sales_data &item);
public:
    Sales_data() = default;
}

其他类作为友元

class Screen {
  // Window_mgr 的成员可以访问Screen类的私有部分
  friend class Window_mgr;
}

其他类的成员函数作为友元

class Screen {
  // Window_mgr::clear 可以访问Screen类的私有部分,必须在Screen类之前被声明
  friend void Window_mgr::clear(ScreenIndex);
}

类型成员

class Screen {
  public:
      // 类型成员
      typedef std::string::size_type pos;
      // 或using pos = std::string::size_type;
  private:
      pos cursor = 0;
      pos height = 0, width = 0;
      std::string contents;
}

令成员函数为内联函数

定义在类中的成员函数是隐式内联的， 而对于定义在类外的成员函数只要在定义的地方使用inline关键字就能声明为内联的，且应该和类定义在一个头文件中。

可变数据成员

有时我们会希望在const成员函数中也能修改某个数据成员，这可以通过给数据成员添加mutable关键字实现，一个mutable数据成员永远不会是const，即使它是const对象的成员。

class Screen {
  public:
      // 统计函数被调用的次数
      void some_member() const;
  private:
      mutable size_t access_ctr;
}

void Screen::some_member() const
  {
    ++access_ctr;
  }

基于const重载

因为非常量版本的函数对于常量版本对象时不可用的，所以我们只能在常量对象上调用const成员函数，另外，虽然非常量对象可以调用常量和非常量版本，但是，显然调用非常量版本是更好的匹配。

class Screen {
  public:
      // 根据对象是否是const重载display 函数
      Screen &display(std::ostream &os)
      { do_display(os); return *this; }
      const Screen &display(std::ostream &os) const
      { do_display(os); return *this; }
  private:
      void do_display(std::ostream &os) const {os << contents;}
}

类的声明

我们可以仅仅声明类而暂时不使用他们

class Screen;

一个类的名字出现后，它就被认为是声明过了,因此，类允许包含指向它自身类型的引用或指针：

class Link_screen {
  Screen window;
  Link_screen *next;
  Link_screen *prev;
};

通过先声明一个类，可以让两个类互相指向：

class Y;

class X {
    Y *y_ptr;
}

class Y {
    X *x_ptr;
}

类的作用域

作用域和定义在类外部的成员

一旦遇到了类名，定义的剩余部分就在类的作用域之内了。如：

void Window_mgr::clear(ScreenIndex i)
{
  Screen &s = screens[i];
  s.contents = string(s.height * s.width, ' ');
}

编译器在处理函数之前已经知道了剩余部分在类的作用域之内了，所以不必专门说明ScreenIndex是Window_mgr类定义的了。但是返回类型在函数名之前出现，所以还要专门指出返回类型在该类中定义。

类的定义过程

类的定义分两步

  1. 编译成员声明
  2. 直到类全部可见才编译函数体

即编译器处理完类中的所有声明后才会处理成员函数的定义，即成员函数可以使用类中定义的任何位置的名字。

上面的两阶段的处理方式只适用于类的成员函数中使用的声明，声明中使用的名字还是需要确保在使用前可见，否则会到外层作用域中取寻找。

类型名的定义一般要出现在类的开始处，因为编译器只会从声明中名字出现的地方的前面去寻找。

类的静态成员

通过加上static关键字声明静态成员，它存在于任何对象之外。静态成员函数也不和任何对象绑定在一起，并且不包含this指针。

class Account {
  public:
      // 成员函数不用通过作用域运算符就能使用静态成员
      void calculate() {amount += amount * interestRate;}
      static double rate() {return interestRate;}
      static void rate(double);
  private:
      std::string owner;
      double amount;
      static double interestRate;
      static double initRate();
};

double r;
r = Account::rate();

Account ac1;
Account *ac2 = &ac1;
r = ac1.rate();
r = ac2->rate();

类外部定义静态成员

当在类外部定义静态成员时，不应该出现static关键字，它只能出现在类内部声明语句。
