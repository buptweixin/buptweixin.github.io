Ipython

魔法方法

  • %run script.py
    在IPython中运行python脚本
  • %whos
    显示当前环境中的变量及其类型和数据
  • %cd
    改变当前工作路径
  • %timeit
    使用标准库中的timeit模块测试函数运行时间
  • %cpaste
    粘贴剪贴板里的内容到IPython中运行，可以包含>>>、In[12]:等标准提示符，以--或Ctrl+D结束
  • %debug
    当程序运行出错时，输入该命令进入ipdb debug模式
  • %alias
    列出模仿UNIX中的一些命令

who查看模块
whos查看变量

赋值操作符

赋值操作符用来绑定(重新绑定)值到名字上或者改变名字对应对象的值。

  • 一个对象可以绑定到多个名字上

    In [1]: a = [1, 2, 3] 
In [2]: b = a
In [3]: a
Out[3]: [1, 2, 3]
In [4]: b
Out[4]: [1, 2, 3]
In [5]: a is b Out[5]: True
In [6]: b[1] = 'hi!' 
In [7]: a
Out[7]: [1, 'hi!', 3]

  • 使用indexing或slices操作在原地改变值

    In [1]: a = [1, 2, 3]
In [3]: a
Out[3]: [1, 2, 3]
In [4]: a = ['a', 'b', 'c'] # Creates another object. In [5]: a
Out[5]: ['a', 'b', 'c']
In [6]: id(a)
Out[6]: 138641676
In [7]: a[:] = [1, 2, 3] # Modifies object in place. In [8]: a
Out[8]: [1, 2, 3]
In [9]: id(a)
Out[9]: 138641676 # Same as in Out[6], yours will differ...
  • mutable对象可以原地改变值，immutable对象一旦创建就不可以改变

包

包是一个包含许多模块的目录，也就是模块的模块。一个包中必须包含一个__init__.py文件来告诉python这是一个包

使用包名.__file__可以得到包所在的路径,如

In [2]: scipy.__file__
Out[2]: '/usr/lib/python2.6/dist-packages/scipy/__init__.pyc'

标准库

os

os模块使你可以方便地使用与操作系统相关的函数

  • 当前路径： os.getcwd()
  • 列出目录下的文件: os.listdir(os.curdir)
  • 新建文件夹：os.mkdir(‘dir’)
  • 重命名文件夹: os.rename(‘foodir’)
  • 删除文件夹: os.rmdir(foodir)
  • 删除文件：os.remove(junk.txt)
os.path
  • a = os.path.abspath(‘junk.txt’):获得绝对路径
  • os.path.split(a):将a的路径和文件名分开存放在元组中
  • os.path.dirname(a):提取a的路径
  • os.path.basename(a):提取a的文件名
  • os.path.splitext(os.path.basename(a)):分开文件名和后缀 “Out[80]: (‘junk’, ‘.txt’)”
  • os.path.exists(‘junk.txt’):判断是否存在
  • os.path.isdir(a):判断是否是路径
  • os.path.expanduser(‘~/local’):把包含~和~user转换为用户目录
  • os.path.join(os.path.expanduser(‘~’), ‘local’, ‘bin’)
  • os.system(‘ls’):运行外部命令；更好的是使用sh模块（Python3中没有了）

遍历文件夹

In [10]: for dirpath, dirnames, filenames in os.walk(os.curdir):
....:   for fp in filenames:
....:     print os.path.abspath(fp)

其中，dirpath为文件夹的路径，dirnames为该文件夹下包含的子目录（相对路径），filenames为dirpath下的非目录文件

sys 模块

和系统有关的python解释器信息，比如什么版本的python，解释器安装位置等

  • sys.platform: 系统
  • sys.version: python解释器版本
  • sys.prefix:解释器安装位置
  • sys.argv: 获得提供给脚本的参数
  • sys.path:模块搜索路径，从PYTHONPATH环境变量获得

In [4]: pickle.load(open('test.pkl','rb')) Out[4]: [1, None, 'Stan']
