C++ primer笔记 — 函数


在头文件中进行函数声明

  1. 自己写的头文件要用#include"xxxx.h"的形式引用
  2. size_t定义在cstddef头文件中，不需要使用using std::size_t

  3. 分离式编译
    假如Chapter6.h包含有函数的声明，fact.cc含有函数的定义，factMain包含使用了该函数的主函数，则使用下列命令编译他们

    $ CC factMain.cc fact.cc #生成factMain.exe 或者 a.out
$ CC factMain.cc fact.cc -o main # 生成main或者main.exe

    以后如果我们修改了其中一个文件，只需要重新编译那个改动了的文件。这就是分离式编译机制，分离式编译机制会产生后缀名为.obj(windows)或.o(UNIX)的文件，含义是该文件包含对象的object code， 接下来编译器负责把对象链接在一起形成可执行文件。

    具体过程如下

    1. $ CC -c factMain.cc # 生成factMain.o
    2. $ CC -c fact.cc #生成fact.o
    3. $ CC factMain.o fact.o #生成factMain.exe 或者 a.out

内联函数和constexpr(常量表达式函数)

inline

对于规模较小的操作可以写入内联(inline)函数中，编译时会自动将内联函数展开。这样做可以从函数字面值而不是阅读源码理解操作的目的、需要修改操作只要修改一处地方就可以了并且利于重用。

inline const string & 
shorterString(const string &s1, const string &s2)
{
  return s1.size() <= s2.size() ? s1 : s2;
}

constexpr

constexpr 函数是指用于常量表达式的函数，它的函数体只能包含不执行任何操作(空语句、类型别名、using声明等)并且有且只有一条返回语句。

constexpr int new_sz() { return 42;}
constexpr size_t scale(size_t cnt) { return new_sz() * cnt;}

如果scale函数的参数是常量表达式的话，它返回值也是常量表达式，正确，否则定义错误。

constexpr隐含的被定义为inline。

inline和constexpr应该放在头文件中

调试帮助

assert预处理宏

定义cassert头文件中，因为它由预处理器管理所以我们可以直接使用预处理命令而不需要使用using声明。

assert一般用于检查不能发生的条件，比如assert(word.size()>threshold);要求所有单词的长度都大于阈值。

NDEBUG预处理变量

assert的状态由NDEBUG控制，如果定义了这个预处理变量则assert什么也不做(需要在引入cassert之前定义)，也可以使用编译器命令定义$CC -D NDEBUG main.C

也可以使用如下方法编写自己的条件调试代码

void print(const int ia[], size_t size) 
{
  #ifndef NDEBUG
    cerr << __func__ <<": array size is " << size << endl;
  #endif
}

其中__func__输出当前调试函数的名字，除此之外其他4个对于程序调试很有用的预定义的名字:

  • `FILE:存放文件名的字符串字面值
  • __LINE__:存放当前行号的整形字面值
    • __TIME__:存放文件编译时间的字符串字面值
    • __DATE__:存放文件日期的字符串字面值

例子

if (word.size() < threshold)
  cerr << "Error: " << __FILE__
       << ": in function " << __func__
       << " at line " << __LINE__ << endl
       << "compile on " << __DATE__
       << "at " << __TIME__ << endl
       << "   word read was \"" << word
       << "\": Length too short" << endl;

函数指针

函数类型是由返回类型和形参共同确定的，如

bool lengthCompare(const string &, const string &);

类型是

bool(const string &, const string &);

所以，可以通过下面的方式声明指向函数的指针

bool (*pf)(const string &, const string &);

上面的扩号不能少，否则变为返回bool型指针的函数。

使用

若把函数名当做值使用，则它会自动转换为指针
pf = lengthCompare;与pf = &lengthCompare;等价，取地址符是可选的

可以直接使用指向函数的指针调用函数，不需要解引用
