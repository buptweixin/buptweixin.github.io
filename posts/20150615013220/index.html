基本内置类型

  1. 执行浮点数运算采用double，因为float的精度往往不够而且执行效率甚至不如double
  2. 指定字面值类型
L'a'  // 宽字符字面值
u8"hi" // utf-8 字符串字面值
42ULL // 无符号整型，类型unsigned long long
1E-3F //单精度浮点数
3.14159L // 扩展精度浮点字面值

变量

  1. string类型定义在标准库中std::string

  2. 列表初始化 (“c++11”)

    int sold = 0;
int sold = {0};
int sold{0};
int sold(0);

    使用{}列表初始化可以防止出现丢失信息的风险，比如将浮点数初始化给新建整型变量，列表初始化不会执行转换操作而普通操作会执行。

  3. 默认初始化
    内置类型如果定义在函数体之外，则会被初始化为0，否则不会被初始化。

  4. 变量的声明和定义的关系
    为了把程序拆开成多个部分(比如，std::cin定义于标准库，但是可以被我们的程序使用),引入了声明和定义的概念。
    一个文件想要使用别处定义的名字需要先声明，而定义除了要规定变量的类型和名字外还需要申请存储空间。

     extern double pi; //声明

复合类型

  1. 引用
    相当于给变量取了另外一个名字
    int val = 1024;
int &refVal = val;
    引用必须被初始化，引用不能绑定在字面值上
  2. 指针

  • 与引用的不同：

    • 引用不是对象，而指针本身就是一个对象，可以对它拷贝和赋值，所以在其声明周期内可以指向几个不同的对象
    • 指针无需在定义的时候赋值。
    int ival = 42;
int *p = &ival; // p中存放ival的地址

  • 空指针
    生成空指针的方法

      int *p1 = nullptr; //等价于int *p = 0; (c++11)
  int *p2 = 0;// 直接将p2初始化为字面值常量0
  // 需要首先#include cstdlib
  int *p3 = NULL; // 尽量避免使用

    把变量直接赋值给指针是错误的，即使其值恰好是0也不行

      int zero = 0;
  pi = zero; //错误

    • void* 指针
      可以存放任意对象的地址

    • 指向指针的指针

      int val = 1024;
int *p = &val;
int **p = &p;

    • 指向指针的引用

      int i = 42;
int *p; //p是一个int型的指针
int *&r = p;  // r是指针p的引用

r = &i; // r引用了指针p，所以给r赋值相当于令p指向i
*r = 0;

      读指向指针的引用应该从用向左读，因为离变量最近的符号对变量有最直接的影响，所以此例中，r最近的&表示r是一个引用，而其他部分表示的是这个引用的类型，即r引用的是一个int指针。

const限定符

const限制变量不能改变，必须初始化

const int i = 3;
const int j; //错误，没有初始化。
  • 默认情况下const只在本文件有效

编译器在编译时，会把所有的const对象都用其具体的值来替代，所以const对象只在本文件有效。

若想要在多个文件中共享const变量，需要声明和定义都使用extern关键字。

// file.cc
extern const int buffSize = fcn();
// file.h
extern const int buffSize;
  • const引用
    const int ci = 1024;
const int &r1 = ci;   //正确
int &r2 = ci; //错误
  • const的引用可能引用非const对象
    int i = 42;
int &r1 = i;
const int &r2 = i;
r1 = 0; // 正确
r2 = 0; //错误

  • 指针和const

    • 指向常量的指针
      不能改变其所指对象的值
      const double pi = 3.14
const double *cptr = π
*cptr = 42; // 错误

    • const指针（常量指针）
      规定指针本身的值不能改变而不是指向的值不能改变，const指针必须初始化

      int errNumb = 0;
int *const curErr = &errNumb; // curErr将一直指向errNumb

const double pi = 3.14;
const double *const pip = π //

      指向常量对象的常量指针，指针本身和所指的值都不能改变。

    • 顶层const和底层const
      顶层const表示指针本身是常量
      底层const表示所指对象是一个常量

    • constexpr变量(c++11)
      constexpr int mf = 20;  //20 是常量表达式
constexpr int limit = mf + 1; //mf + 1是常量表达式
constexpr int sz = size(); //只有当size是一个constexpr常量表示式时才是一条正确的表达式

编写自己的头文件

为了确保各个文件中的类的定义一致，类通常被定义在头文件中。类的头文件名应该与类一样。
头文件通常包含那些只能被定义一次的实体，比如类、const和constexpr变量。
头文件通常用到其他头文件的功能，比如Sales_item.h有一个String成员，所以必须包含string.h,而使用Sales_item类的程序需要对这个string操作所以需要再次包含String.h,这样就包含了两次string.h，为了解决这个冲突，需要用到预处理技术

预处理器

预处理器是编译之前执行的一段程序，当它看到#include时，就会用指定的头文件替代它；
还会用到的另一种预处理器功能是头文件保护符：
#define把一个名字设定为预处理变量，另外连个变量检测某个预定的预处理变量是否已经定义：#ifdef当且仅当变量已定义时为真，#ifndef当且仅当变量没有定义时为真，一旦检查结果为真，则会一直执行到#endif

  #ifndef SALES_DATA_H
  #define SALES_DATA_H
  #include 
  struct Sales_data {
    std::string bookNo;
    unsigned units_sold = 0;
    ...
    };
  #endif

这样，第一次包含Sales_data.h时，检查结果为真，会执行到#endif,后面如果再一次包含，则检查结果为假
