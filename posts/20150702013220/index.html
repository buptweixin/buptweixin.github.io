顺序容器概述

容器 随机访问 顺序访问 特点
vector √ √ 尾部之外的位置插入或删除可能很慢
deque √ √ 在头尾位置插入删除很快
list × 双向 在任何位置进行插入删除操作速度都很快，额外内存开销很大
forward_list × 单向 在链表任何位置删除插入速度都很快，额外内存开销很大，和最好的手写单向列表相当的性能，没有size操作
array √ √ 不能添加或删除元素
string 与vector相似，但专门用来保存字符

每个容器都定义在类型名相同的头文件中，且均为模板类

选择容器技巧

  1. 通常，选择vector是最好的选择，除非你有很好的理由选择其他容器。
  2. 有很多小元素且空间开销很重要，则不要使用list或forward_list
  3. 如果要求随机访问，应使用vector或deque
  4. 如果要求在中间插入元素，应该使用list或forward_list
  5. 如果程序需要头尾插入元素但不会从中间插入，则选择deque
  6. 如果程序只有在输入时需要向中间位置插入元素，随后需要随机访问元素，则
    • 确定是否真的需要中间插入，比如顺序读入到vector，之后调用sort排序，从而避免中间读入
    • 必须中间插入时，在输入阶段使用list，输入完成时将list拷贝到vector中
