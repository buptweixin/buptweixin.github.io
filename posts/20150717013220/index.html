C++ Primer — STL:泛型算法

概述

大多数泛型算法的头文件在algorithm头文件中，numeric头文件中还定义了一组数值泛型算法。

一般来书泛型算法操作的都是迭代器，如查找算法find:

auto result = find(vec.cbegin(), vec.cend(), val);

注意，若valfind返回的是指针.

注意，算法本身不会执行容器的操作，它们只会运行在容器上，所以算法永远不会改变底层容器的大小。要实现这样的效果，可以使用一类特殊的迭代器—插入器

初始泛型算法

  • 只读算法

    • find

    • count

    • accumulate

      定义在头文件numeric中

      int sum = accumulate(vec.cbegin(), vec.cend(), 0);

      前两个参数是迭代器范围，第三个参数是和的初始值

    • equal

      equal(roster1.cbegin(), roster1.cend(), roster2.cbegin());

  • 写容器元素算法

    • fill

      fill(vec.begin(), vec.end(), 0);//将每个元素置为0

    • fill_n

      fill_n(vec.begin(), vec.size(), 0);//将所有元素置为0
fill_n(dest, n ,val);//将dest之后的n个元素置为val

      ​ 不能对空的vector使用fill

    • back_inserter

      back_inserter是一种向容器中添加元素的迭代器，定义在iterator头文件中，它接受一个指向容器的引用返回一个与该容器绑定的插入迭代器。当我们通过它赋值时，赋值运算会调用push_back将一个具体给定元素添加到容器中

      vector<int> ivec, vec;//空的容器
auto it = back_inserter(ivec);
*it = 42; //现在容器中有了一个新元素42
fill_n(back_inserter(vec), 10, 0);

    • 拷贝算法

      接受三个参数，前两个指定源范围，最后一个指定目的的起始位置，目的序列至少要有和源序列一样多的元素

      许多算法都提供了拷贝版本，比如replace

      replace(ilst.begin(), ilst.end(), 0, 42);
replace_copy(ilst.begin(), ilst.end(), back_inserter(ivec), 0, 42);

      ​

  • 重排容器元素的算法

    • sort排序

      接受两个参数，表示要排序的元素的范围

    • unique将相邻的重复项消除

      返回一个指向不重复范围末尾的迭代器，容器的大小并没有改变，只是元素的顺序改变了。要删除元素必须使用容器操作，比如erase

定制操作

有些时候我们需要定制sort等算法的行为，比如不使用字典序而是使用字符串的长度作为比较依据，这时候就需要添加谓词操作。

  1. 向算法传递函数

    1. 谓词

      谓词是可以调用的表达式，分为一元谓词和二元谓词，分别能接受一个和两个参数

      bool isShorter(const string &s1, const string &s2)
{
  return s1.size() < s2.size();
}
sort(words.begin(), words.end(), isShorter);

    2. 排序算法

      c++标准库定义了stable_sort算法，主要的不同时它不改变相同元素的相对位置。

  2. lambda表达式

    ​ 可以理解成未命名的内联函数

    1. 形式

      [capture list](parameterr list) -> return type {function body}

      capture list为捕获列表，用于获得函数中的非静态变量，函数外部和函数内部静态变量默认可见，这个列表通常为空。parameter list参数列表不能有默认参数；return type和function body和普通函数一样。

      可以忽略参数列表和返回类型，但是必须包括捕获列表和函数体。

      auto f = [] { return 42;};
cout << f() << endl; //和普通函数一样调用

    2. 使用捕获列表

      [sz](const string &a)
    {return a.size() >= sz;};

  3. for_each算法

    for_each算法接受一个可调用对象，并对输入序列中每个元素调用此对象。

    ​

  4. 值捕获和引用捕获

    默认是值捕获，捕获列表使用[&sz]是引用捕获。

  5. 引用捕获

    对于函数内静态变量和外部变量，需要显式使用=和&表明它们是值捕获还是引用捕获

    [=](const string &s)
    {return s.size() > sz;};
[&, c](const string &s){os << s << c;}; //os隐式引用捕获，c显式值捕获

  6. 可变lambda

    默认情况下值捕获复制的值不能改变，如果需要改变它，必须在参数列表首加上mutable,而引用捕获是否能够修改需要根据被引用对象是否是const决定。

  7. 尾置返回类型

    上例中我们无需指明返回类型，应为它只有一个返回语句，所以lambda可以推断出返回类型

    这种情况下编译器推断返回类型为void，实际上返回了int

    上面的例子使用了尾置返回类型指明了要返回什么类型。

  8. 参数绑定

    可以使用bind进行参数绑定改变函数参数数量或参数相对位置

    bind定义在头文件functional中，其形式如下：

    auto newCallable = bind(callable, arg_list);

    arg_list可能包含形如_n的名字，它定义在std::placeholders的命名空间中，n代表参数在newCallable中出现位置，比如

    //f有5个参数
auto g = bind(f, a, b, _2, c, _1);
//它将
g(X, Y);
//映射为
f(a, b, Y, c, X);

    有时候我们希望bind中非占位符参数以引用的方式传递或要绑定的参数无法拷贝，比如ostream，则需要使用functional头文件中的ref函数

    for_each(words.begin(), words.end(),
         bind(print, ref(os), _1, " "));

    标准库中还有cref函数，生成一个const 引用类。

    ​

再探迭代器

标准库头文件iterator中定义了下列迭代器

  • 插入迭代器：用来向容器插入元素

    • back_inserter:创建一个使用push_back的迭代器
    • front_inserter:创建一个使用push_front的迭代器
    • inserter:创建一个使insert的迭代器

  • 流迭代器：绑定到输入输出流上，用来遍历他们

    • istream_iterator

    • ostream_iterator

  • 反向迭代器：这些迭代器向前移动而不是向后移动，除了forward_list其他容器都有

  • 移动迭代器：移动元素而不是拷贝他们，见13.6.2节
