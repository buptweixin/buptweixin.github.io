LeetCode 4 Median of Two Sorted Arrays

[LeetCode]4:Median of Two Sorted Arrays

题目：

There are two sorted arrays nums1 and nums2 of size m and n respectively.

Find the median of the two sorted arrays. The overall run time complexity should be O(log (m+n)).

Example 1:

nums1 = [1, 3]
nums2 = [2]

The median is 2.0

Example 2:

nums1 = [1, 2]
nums2 = [3, 4]

The median is (2 + 3)/2 = 2.5

解题思路

中间元素所在位置的特点是:

  1. 左边元素个数等于右边元素个数
  2. 左边最大元素不大于右边最小元素

假设将nums1分为nums1[:i]和nums[i:],nums2分成nums2[:j]和nums2[j:],如果i+j（左半部分长度） == (m-i) + (n-j) if((m+n)%2 ==0)或 (m - i) + (n - j) + 1 if ((m+2)%2 == 1)(右半部分长度) ,则满足上面的条件一,另外 0<i<m, 0<j<n,j=(m+n+1)/2-i 推出n>=m,所以如果nums1.size() > nums2.size()则将他们调换；

接下来，判断是否满足条件二，因为两个数组已经排过序了，即nums1[i-1] < nums[i], nums2[j-1]<nums2[j],只需要考虑nums1[i]是否大于nums2[j-1],nums[j]是否大于nums[i-1]

class Solution {
public:
    double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) {
        int m = nums1.size(), n = nums2.size();
        if (m > n) return findMedianSortedArrays(nums2, nums1);
        int i, j, imin = 0, imax = m, half = (m + n + 1) / 2;
        while (imin <= imax) {
            i = (imin & imax) + ((imin ^ imax) >> 1);
            j = half - i;
            if (i > 0 && j < n && nums1[i - 1] > nums2[j]) imax = i - 1;
            else if (j > 0 && i < m && nums2[j - 1] > nums1[i]) imin = i + 1;
            else break;
        }
        int num1;
        if (!i) num1 = nums2[j - 1];
        else if (!j) num1 = nums1[i - 1]; 
        else num1 = max(nums1[i - 1], nums2[j - 1]);
        if ((m + n) & 1) return num1;
        int num2;
        if (i == m) num2 = nums2[j];
        else if (j == n) num2 = nums1[i];
        else num2 = min(nums1[i], nums2[j]);
        return (num1 + num2) / 2.0;
    }
};

C++ primer笔记 -- 标准库--顺序容器

SLT — 顺序容器

  • 顺序容器

    顺序容器中的顺序与其加入容器的时的位置相对应

  • 关联容器

    关联容器中元素的位置由元素相关联的关键字决定

顺序容器概述

容器 随机访问 顺序访问 特点
vector √ √ 尾部之外的位置插入或删除可能很慢
deque √ √ 在头尾位置插入删除很快
list × 双向 在任何位置进行插入删除操作速度都很快，额外内存开销很大
forward_list × 单向 在链表任何位置删除插入速度都很快，额外内存开销很大，和最好的手写单向列表相当的性能，没有size操作
array √ √ 不能添加或删除元素
string 与vector相似，但专门用来保存字符

每个容器都定义在类型名相同的头文件中，且均为模板类

选择容器技巧

  1. 通常，选择vector是最好的选择，除非你有很好的理由选择其他容器。
  2. 有很多小元素且空间开销很重要，则不要使用list或forward_list
  3. 如果要求随机访问，应使用vector或deque
  4. 如果要求在中间插入元素，应该使用list或forward_list
  5. 如果程序需要头尾插入元素但不会从中间插入，则选择deque
  6. 如果程序只有在输入时需要向中间位置插入元素，随后需要随机访问元素，则
    • 确定是否真的需要中间插入，比如顺序读入到vector，之后调用sort排序，从而避免中间读入
    • 必须中间插入时，在输入阶段使用list，输入完成时将list拷贝到vector中
